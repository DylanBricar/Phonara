(branch `main`). Each plan is **self-contained**: an executor with zero context
from the audit can run it using only the plan file and the repo. Read the plan

app forked from Handy). The fork's defining addition is **LLM post-processing of
transcripts**, which is where most findings cluster (security, dead code, docs,

## Findings considered and rejected

So they aren't re-audited next run:

- **"Corrupted settings crash the app on startup"** — REJECTED (misread).
  `get_settings` (`settings.rs:1107`) recovers from a malformed store via
  `unwrap_or_else(|_| get_default_settings())`; the remaining `.unwrap()`s
  serialize known-good default structs and effectively never fail.
- **External paste-script path has no validation** (`clipboard.rs:504`) —
  REJECTED (by-design). It's an explicit user-configured `PasteMethod::ExternalScript`
  feature (analogous to configuring `$EDITOR`). `Command::new(path).arg(text)`
  does not invoke a shell, so there's no injection; path validation would be
  minor hardening at best.
- **tar.gz model-extraction path traversal** (`managers/model.rs`) — REJECTED
  (low signal). The `tar` crate rejects `..`/absolute entries by default.
- **Prompt-injection via transcribed text into the LLM prompt** — NOTED, not
  planned. Inherent to any "speak → LLM" tool; the threat model is the user's own
  speech. A structured-prompt delimiter is a reasonable future hardening but not a
  ranked finding.
- **`gemini_client.rs` empty-response bug** — REJECTED as a live bug; the file is
  dead code (covered by plan 002, which deletes it).

## Deferred (real, but not turned into a plan this round)

- **`settings.rs` is a 1168-line god module** (16 enums + ~40 default fns +

  boundary file. It is **intentionally not planned yet**: it should follow plan
  011 (so characterization tests exist before the split) and warrants its own
  design pass rather than a blind executor handoff. Re-evaluate after 011 lands.
